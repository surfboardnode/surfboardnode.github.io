一、Quantumult X 是什么？它的定位远超代理工具

Quantumult X（以下简称QX）不仅仅是一个普通的代理客户端，它是一套网络管理解决方案。其核心定位是——为用户提供完整、透明、可控的网络请求处理逻辑。

在安卓版本中，QX保留了大部分iOS核心功能，并加入了一些平台特性，让安卓用户也能享受到高级代理技术带来的便利与自由。

核心能力概览：

  • 支持多协议代理：包括 HTTP、HTTPS、SOCKS5、Shadowsocks、Vmess 等。

  • 灵活的规则配置系统：支持根据域名、IP、关键词等设定行为逻辑。

  • 自定义 DNS 服务：提高解析速度，绕过DNS污染。

  • 分流与重定向支持：可将不同应用或网站的流量分配给不同的节点或策略。

  • 实时流量监控和日志记录：网络行为透明可见，方便排错与优化。

  • 自动更新订阅与规则集：支持外部订阅链接，轻松实现节点批量管理。

在功能层级上，QX几乎达到了商用级别的网络管理水平，却以极简的方式呈现给终端用户，可谓“简洁中见锋芒”。

三、Quantumult X 配置实战：从零开始掌握核心能力

真正让Quantumult X发挥价值的，是它强大的配置系统。以下将从基础代理设定开始，一步步讲解如何打造属于你的QX网络规则体系。

1. 添加代理服务器节点（节点来源）

要实现科学上网，首先需要代理节点。节点来源一般有三种：

  • 购买付费节点订阅（推荐）：如Clash、V2ray、Trojan等服务商提供的订阅链接，稳定、安全。

  • 使用免费分享节点：不稳定，安全性低，适用于测试和临时访问。

  • 自建节点：成本高但可控，适合高阶用户。

导入方式：

  • 打开Quantumult X主界面，点击右上角“+”，选择“通过订阅链接导入”；

  • 粘贴你的订阅链接；

  • 导入后会自动更新节点列表。

2. 设置DNS服务器

DNS 是网络访问中的第一步，决定了你能否顺利访问目标网站。

推荐配置：

  • Cloudflare DNS：1.1.1.1 / 1.0.0.1

  • Google DNS：8.8.8.8 / 8.8.4.4

在设置中手动输入DNS地址，建议开启 DNS-over-HTTPS 或 DNS-over-TLS，防止DNS污染。

3. 自定义规则分流

这是Quantumult X最有价值的部分。你可以将如下流量自动分流：

  • 国内网站 → 直连

  • 国际网站（如Twitter、YouTube）→ 走代理

  • 广告域名 → 拦截

  • 游戏类请求 → 分配特定节点（如低延迟日本节点）

示例规则：

ini
[filter]
IP-CIDR,192.168.0.0/16,DIRECT
DOMAIN-SUFFIX,twitter.com,Proxy
DOMAIN-KEYWORD,google,Proxy
GEOIP,CN,DIRECT
FINAL,Proxy

规则执行顺序从上到下，最后一条是“兜底规则”，未命中其他规则的流量将默认走 Proxy。

4. 快速启用和切换代理

在主界面中可快捷开启/关闭代理服务，还可在不同代理策略之间切换。

建议模式：

  • 工作时：规则分流

  • 下载大文件或看视频时：全局代理

  • 网络调试时：直连或记录日志模式

FAQ

VMess 使用 TLS+WebSocket 的好处是什么？
TLS+WebSocket 可伪装成正常 HTTPS 流量，增强抗封锁能力和隐蔽性。结合域名伪装和策略组使用，可在高封锁网络环境下保持安全、稳定和高速的网络访问体验。
WinXray 节点延迟检测如何优化视频访问？
WinXray 定期测速节点延迟和丢包率，并结合策略组自动选择最佳节点。保证视频流量通过低延迟节点，减少缓冲和卡顿，提高播放流畅度和网络体验。
Sing-box 可以替换 Clash 配置文件吗？
Sing-box 支持直接解析 Clash YAML 配置，并转换为自身 JSON 格式。节点、策略组和规则基本兼容，但部分自定义脚本可能需要手动调整才能完全使用。
PassWall2 如何自动切换最优节点？
PassWall2 会定期测试节点延迟和丢包率，并结合策略组自动切换到最优节点。无需手动干预即可保证网页、视频和游戏流量高速稳定访问，优化整体网络体验。
VLess 为什么被认为是未来趋势？
VLess 去掉了内置加密层，改为依赖外部 TLS 进行安全通信，这使其更轻量化且更灵活。它能与 Hysteria、Reality 等现代传输方式结合使用，适应性更强，是取代 Vmess 的新标准。
QuantumultX 如何实现按协议类型分流？
QuantumultX 可根据 TCP、UDP 或特定应用协议分流流量，结合策略组选择节点。这样可以为游戏、视频或广告屏蔽等应用提供最佳网络体验，提高访问速度和稳定性。
V2Ray 的 WebSocket + TLS 如何防止流量被检测？
WebSocket + TLS 可将 V2Ray 流量伪装成正常 HTTPS 请求，隐藏真实数据特征，配合域名伪装和策略组使用，可有效绕过封锁，提高节点安全性和连接稳定性。
Nekoray 与 Nekobox 有什么区别？
Nekoray 是基于 sing-box 的多平台代理客户端，支持 Windows、Linux 等系统；而 Nekobox 则是为 Android 设计的移动版本。两者共享相同的核心逻辑，均支持 VMess、VLess、Trojan 等协议，适合跨设备同步使用。
Hysteria2 的 QUIC 协议如何降低延迟？
QUIC 协议基于 UDP，多路复用和拥塞控制优化，可减少握手延迟和包重传次数。Hysteria2 利用 QUIC 的特性，使高延迟或丢包网络下依然保持低延迟和高稳定性。
Hiddify Next 如何设置用户带宽限制？
管理员可为每个用户设置上传和下载带宽上限。超出限制可自动降速或阻止访问，保证服务器资源合理分配，避免单用户占用过多带宽，提高整体公平性和使用效率。
